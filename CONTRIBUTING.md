# Внесение вклада в JW Library Backup Merger

Спасибо за интерес к проекту JW Library Backup Merger! Мы рады сотрудничеству и приветствуем ваши предложения.

## С чего начать

1. **Форкните** репозиторий
2. **Клонируйте** свой форк на локальную машину
3. Создайте **новую ветку** для вашей функции (`git checkout -b feature/NewFeature`)
4. **Внесите изменения** в код
5. **Зафиксируйте** изменения (`git commit -m 'Add NewFeature'`)
6. **Отправьте** ветку в ваш форк (`git push origin feature/NewFeature`)
7. Откройте **Pull Request**

## Типы вклада

Мы ценим любой тип вклада, включая:

- Исправления ошибок
- Новые функции
- Обновления документации
- Исправления опечаток
- Улучшения производительности
- Расширение тестов

## Требования к коду

- Пишите чистый, легко читаемый Python-код
- Добавляйте комментарии к сложным участкам кода
- Следуйте соглашениям об именовании PEP 8
- Обновляйте документацию при необходимости

## Тестирование

Перед отправкой изменений убедитесь, что:

- Ваш код работает корректно на тестовых данных
- Вы запускали основной скрипт и проверили его функциональность
- Код не вызывает ошибок в различных сценариях использования

## Сообщения коммитов

Придерживайтесь следующего формата:

```
type(scope): description

body (optional)

footer (optional)
```

Пример:
```
feat(merger): добавлена поддержка новых форматов тегов

- Обновлён алгоритм обработки тегов
- Добавлена защита от ошибок структуры
```

Типы:
- `feat` – новая функция
- `fix` – исправление ошибки
- `docs` – изменения документации
- `test` – изменения в тестах
- `refactor` – рефакторинг кода
- `style` – изменения стиля кода
- `chore` – прочие изменения

## Сообщения об ошибках

При создании Issue:

1. **Проверьте**, не описана ли уже похожая проблема
2. **Опишите** проблему четко и подробно
3. **Укажите** шаги для воспроизведения
4. **Приложите** возможные ошибки или скриншоты
5. **Опишите** вашу систему и версию Python

## Запросы на функции

При создании запроса на новую функцию:

1. **Объясните**, какую проблему решает ваша идея
2. **Опишите**, как должна работать новая функция
3. **Обсудите** возможные реализации
4. **Учтите**, как функция вписывается в архитектуру проекта

## Кодекс поведения
